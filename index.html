命名

命名常常被认为是编程中的细节问题,其重要性往往被低估.而所谓的工匠精神,往往就是体现在细节之处.一个名字虽然并不影响程序的执行,但是却对代码的表达力和可读性有着重要的影响.

在程序员的工作中,大部分的时间都在阅读和理解代码,好的命名能够让代码的概念清晰,增强代码的表达力;词不达意的命名会破坏我们思考的连贯性,分散有限的注意力.

命名的力量

无论是对于人名,还是企业名产品名,命名都有着巨大的力量

命名其实很难

起名字这件事看似不难,但是要经过深思熟虑,取出名副其实.表达性好的名字并不是一件很容易的事

命名为什么难呢?因为命名的过程本身就是一个抽象和思考的过程,在工作中,当我们不能给一个模块,一个对象,一个函数,甚至一个变量找到合适的名称的时候,往往说明我们对问题的理解还不够透彻,而需要重新去挖掘问题的本质,对问题域进行重新分析和抽象,有时还要设计和重构代码.因此,好的名字是我们写出好代码的基础.

有意义的命名

代码即是文档,可读性好的代码应该有一定的自明性,也就是不借助注释和文档,代码本身就能显性化的表达开发者的意图.这种自明性在很大的程度上依赖于我们对问题域的理解,以及命名是否合理.

通常如果我们无法想出一个合适的名字,很可能意味着代码”坏味道”.设计有问题.这时候可以思考一下,是不是一个方法里实现了太多的功能?或者类的封装内聚性不够?又或者是我们对问题的理解还不够透彻?需要获取更多的信息?

变量名

变量名应该是名词,能够正确的描述业务,有表达力.如果一个变量名需要注释来补充说明,那么很有可能说明明明就有问题

函数名

函数的命名要体现做什么,而不是怎么做.假如我们将雇员的信息存储在一个栈中,现在需要从栈中获取最近存储的一个雇员信息,那么getLatestEmployee()就比popRecord()要好,因为栈数据结构是底层实现细节,命名应该提升抽象层次,体现业务语义.合理的命名可以是我们省略掉记住”出栈”的脑力步骤,我们只需要简单地说”取最近雇员的信息”.

类名

类是面向对象中最重要的概念之一,是一组数据和操作的封装.对于一个应用系统,我们可以将类分为两大类:实体类和辅助类.

实体类承载了核心业务数据和核心业务逻辑,其命名要充分体现业务语义,并在团队内达成共识,如Customer,Bank和Employee等.

辅助类是辅佐实体类一起完成业务逻辑的,其命名要能够通过后缀来体现功能.例如,用来为Customer做控制路由的控制类CustomerController/提供Customer服务的服务类CustomerService,获取数据存储的仓储类CustomerRepository.

创建我的第一篇博客文章

第一步 环境搭建

1.安装Node.js

https://nodejs.org

2.安装Git

https://git-scm.com/downloads

第二步 安装Hexo

  • win系统打开终端
    • win健 + R ,输入cmd
    • 开始菜单鼠标右键,选择PowerShell(尽量以管理员模式下进行)
    • 也可以打开VS Code,按Ctrl + ` 打开终端

1.通过npm包管理工具指定网址安装cnpm

2.利用cnpm来安装全局hexo

3.验证hexo安装是否成功

第三步 创建博客

1.新建文件夹mkdir + 文件名, 文件名不需要加”

mkdir blog 就创建了文件夹名为blog的文件夹

如果在搭建博客过程中出现任何问题,大不了把文件夹删掉重来,无需担心

2.进入创建之后的文件夹

3.初始化博客(新建一个网站)

4.出现Start blogging with Hexo! 证明初始化成功

5.dir命令可以查看当前目录下的所有文件

6.hexo server命令启动服务器(可简写为hexo s)

7.可打开浏览器,在网址栏输入localhost:4000,回车键即可查看

第四步 新建文章

1.hexo new 文章名 新建一篇文章 (注意文章名需要加”)

2.命令cd source/_posts 进入资源目录

3.dir 可查看目录下新建的文章

第五步 编辑文章

1.可使用编辑器VS Code等, Markdown软件Typora等打开后缀名为.md的文件

2.编辑文章内容参考Markdown语法即可

3.编辑完成之后返回到主目录,也就是一开始新建的文件夹

4.hexo clean 清除缓存文件

5.hexo generate 生成静态文件(可简写为hexo g)

6.hexo server 启动服务器(可简写为hexo s)

7.本地搭建已经完成

第六步 部署到GitHub

1.首先登录GitHub个人账号

2.创建新的存储库

3.部署个人博客github地址

4.创建存储库

5.创建完成

6.创建git部署的插件

7.使用VS Code打开_config.yml文件

8.编辑如下内容

9.部署到远端

10.输入git账号和密码
